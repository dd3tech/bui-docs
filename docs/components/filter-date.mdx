# <HeaderDocCustom title="Filter Date" pathUrl="components-filterdate--filter-date" />


The Filter Date component allows users to filter and select dates intuitively in web and mobile applications.

<br />

##### <span name="floating-nav">Imports</span>

The first alternative is recommended since they can reduce the application bundle

<WindowEditor codeString="import FilterDate from 'dd360-ds/FilterDate'" />
<WindowEditor codeString="import { FilterDate } from 'dd360-ds'" />

##### <span name="floating-nav">Usage</span>

The prop onApply of type function is mandatory, this function returns an object containing the month and year values selected by the user.

<Playground>
  <FilterDateCustom />
</Playground>

The code snippet below shows how to use the FilterDate component:

<WindowEditor
    codeString={`import { FilterDate } from 'dd360-ds''

<FilterDate
  onApply={() => undefined}
/>
`}/>


##### <span name="floating-nav">Customization</span>

<span className="font-semibold" >The FilterDate component can be customized using the following props:</span>
 - title: The title displayed when the component is opened.
 - textApplyBtn: The text displayed on the apply button.
 - textResetBtn: The text displayed on the reset button.
 - language: The language used for the component. Accepts 'en' (English) or 'es' (Spanish) as values.

Example customization:

<Playground>
  <FilterDateCustom
    title="Select date"
    textApplyBtn="Apply date"
    textResetBtn="Reset date"
    language="en"
  />
</Playground>

The code snippet below shows how to customize the FilterDate component:

<WindowEditor
    codeString={`import { FilterDate } from 'dd360-ds'

<FilterDate
  onApply={() => undefined}
  title="Select date"
  textApplyBtn="Apply date"
  textResetBtn="Reset date"
  language="en"
/>
`}/>

##### <span name="floating-nav">Support Server Side Rendering</span>

For applications that use server-side rendering, you can ensure that the FilterDate component is fully rendered before displaying it by using the domLoaded state variable. 
This helps avoid any issues that may arise due to server-side rendering.

Example usage:

<WindowEditor
    codeString={`import { FilterDate } from 'dd360-ds'

const FilterComponent = () => {
  const [domLoaded, setDomLoaded] = useState<boolean>(false)

  useEffect(() => {
    setDomLoaded(true)
  }, [])

  return (
    <>
      {domLoaded && (
         <FilterDate
            onApply={() => undefined}
          />
      )}
    </>
  )
}

export default FilterComponent
 `}/>
